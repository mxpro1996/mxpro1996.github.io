1. Linux文件权限概述

2. 权限位的组成

Linux文件权限位通常由10个字符组成，格式如下：

-rwxr-xr-x

这10个字符可以分解为以下部分：

  • 第一个字符：文件类型标识符

    • - 表示普通文件
    • d 表示目录
    • l 表示符号链接
    • 其他字符如 b、c 等表示块设备文件和字符设备文件等

  • 接下来的9个字符：分为三组，每组3个字符，分别表示文件所有者、所属用户组和其他用户的权限。每组三个字符代表以下含义：

    • 第一个字符：读取权限 (r)，如果有读取权限则为 r，否则为 -。
    • 第二个字符：写入权限 (w)，如果有写入权限则为 w，否则为 -。
    • 第三个字符：执行权限 (x)，如果有执行权限则为 x，否则为 -。

因此，权限位的详细含义如下：

  • 所有者权限（Owner Permissions）
  • 组权限（Group Permissions）
  • 其他用户权限（Others Permissions）

3. 数字表示法

Linux中的权限也可以用数字表示，称为八进制表示法。每一组权限（读、写、执行）都由一个数字表示：

  • 读 (r) = 4
  • 写 (w) = 2
  • 执行 (x) = 1

三种权限的组合加起来形成一个数字，表示文件所有者、组和其他用户的权限。例如：

  • rwx = 4 + 2 + 1 = 7
  • rw- = 4 + 2 + 0 = 6
  • r-- = 4 + 0 + 0 = 4

因此，整个文件权限表示如 -rwxr-xr-x 可以用数字 755 来表示，其中：

  • 所有者权限为 7（rwx）
  • 组权限为 5（r-x）
  • 其他用户权限为 5（r-x）

4. 权限管理命令

4.1 查看文件权限

要查看文件的权限，可以使用 ls -l 命令：

ls -l filename

输出示例：

-rwxr-xr-x 1 user group 12345 Oct 17 10:00 filename

4.2 修改文件权限

要修改文件的权限，可以使用 chmod 命令。

  • 使用符号表示法：
chmod u+x filename  # 给文件所有者添加执行权限
chmod g-w filename  # 移除文件所属组的写入权限
  • 使用数字表示法：
chmod 755 filename  # 设置文件权限为 rwxr-xr-x

4.3 修改文件所有者和组

要修改文件的所有者和所属组，可以使用 chown 命令：

chown user:group filename  # 设置文件的所有者为 user，所属组为 group

4.4 设置特殊权限

Linux支持三种特殊权限：SUID、SGID和Sticky Bit。

  • SUID（Set User ID）：文件被执行时，以文件所有者的身份执行，通常用于需要管理员权限的程序。设置方式：
chmod u+s filename
  • SGID（Set Group ID）：文件被执行时，以文件所属组的身份执行。对于目录来说，SGID会使得新创建的文件继承目录的组权限。设置方式：
chmod g+s filename
  • Sticky Bit：通常用于目录，只有文件的所有者可以删除或重命名该目录中的文件。设置方式：
chmod +t directory_name
